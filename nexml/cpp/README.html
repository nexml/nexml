The C++ bindings are generated from the xsd schema files using tools and headers from CodeSynthesis, and are then compiled with something like the driver.cxx file (kindly provided by Boris Kolpackov of CodeSynthesis) to yield an executable that processes nexml instance documents.

Prerequisites

  • xerces-c: we need to include its headers during compiling and libxerces during linking. For example, if XERCES_HOME=~/xerces-c_2_8_0-x86-macosx-gcc_4_0, compile with -I$XERCES_HOME/include and link with -L$XERCES_HOME/lib -lxerces-c.
  • The 3.1.0 release of CodeSynthesis xsd. We need the xsd executable to generated source and headers from schema files, and we need to include the headers in the release when compiling this generated code. For example, if XSD_HOME=~/xsd-3.1.0.a6-i686-macosx, then generate code using the executable $XSD_HOME/bin/xsd and compile with -I$XSD_HOME/libxsd.
  • GCC: I (RAV) use the g++ from gcc version 4.0.1 (Apple Computer, Inc. build 5367), I assume any recent version is fine

Building

Both the makefile and the build.sh shell script show how this is done - but they need to be tweaked for your own configuration.
